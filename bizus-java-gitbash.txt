################# INSTALAR JDK #################
Instalar o JDK Java Development Kit no site da Oracle ou do OpenJDK
No GitBash, digite
curl -L -O https://github.com/adoptium/temurin17-binaries/releases/download/jdk-17.0.1+12/OpenJDK17U-jdk_x64_windows_hotspot_17.0.1_12.zip
unzip OpenJDK17U-jdk_x64_windows_hotspot_17.0.1_12.zip -d ~/jdk-17
nano ~/.bashrc

Acrescente no final do esport (utilizando as setas)

export PATH="$JAVA_HOME/bin:$PATH"

Pressione Ctrl + O (isso irá abrir a opção de "Write Out" para salvar o arquivo)
Pressione Enter para confirmar o nome do arquivo e salvar (não precisa nomear o arquivo)
Pressione Ctrl + x (para sair do nano)

source ~/.bashrc (Execute o seguinte comando no Git Bash para aplicar as mudanças:)
java -version

################# PROJETO SPRINGBOOT #################

acessar site https://start.spring.io/

Preencher os campos do site:
Para preencher os campos no Spring Initializr, aqui estão algumas orientações com base no seu projeto:

Project: Escolha Maven como a ferramenta de construção, pois você está usando Maven no seu projeto atual.

Language: Selecione Java, já que você está desenvolvendo em Java.

Spring Boot: Escolha a versão mais recente estável do Spring Boot disponível. Por exemplo, você pode escolher a versão 2.5.8 (que é a última versão estável no momento).

Project Metadata:

_ Group: Defina o grupo do seu projeto. Por exemplo, com.example é comum para projetos de demonstração.
_ Artifact: Este é o nome do seu artefato (arquivo JAR ou WAR). Por exemplo, Java-Servidor-HTTP-POST.
_ Name: Nome descritivo do projeto, como Java Servidor HTTP POST.
_ Description: Uma breve descrição do seu projeto, por exemplo, "Projeto de servidor HTTP POST em Java usando Spring Boot".
_ Package name: O nome do pacote principal do seu projeto, como com.example.javahttpserver.
_ Packaging: Escolha Jar ou War, dependendo de como você deseja empacotar sua aplicação. Geralmente, para aplicações standalone, Jar é mais comum.
_ Java: Escolha a versão do Java que você está usando. Parece que você está usando o Java 17, então selecione a versão correspondente.
_ Dependencies: Aqui você adicionará as dependências específicas do Spring que deseja incluir no seu projeto. Para um servidor HTTP POST simples, você precisará pelo menos da dependência Spring Web.
Depois de preencher todos os campos conforme necessário, clique no botão GENERATE para baixar um arquivo ZIP contendo o projeto inicializado com suas configurações. Você pode então descompactar o arquivo ZIP no diretório do seu projeto atual e importá-lo na sua IDE para começar a trabalhar.

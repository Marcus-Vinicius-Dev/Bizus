*AMBIENTE TERMINAL GIT BASH

Instalar o JDK Java Development Kit no site da Oracle ou do OpenJDK
No GitBash, digite
curl -L -O https://github.com/adoptium/temurin17-binaries/releases/download/jdk-17.0.1+12/OpenJDK17U-jdk_x64_windows_hotspot_17.0.1_12.zip
unzip OpenJDK17U-jdk_x64_windows_hotspot_17.0.1_12.zip -d ~/jdk-17
nano ~/.bashrc

Acrescente no final do esport (utilizando as setas)
export JAVA_HOME="$HOME/jdk-17/jdk-17.0.1+12"
export PATH="$JAVA_HOME/bin:$PATH"

Pressione Ctrl + O (isso irá abrir a opção de "Write Out" para salvar o arquivo)
Pressione Enter para confirmar o nome do arquivo e salvar (não precisa nomear o arquivo)

source ~/.bashrc (Execute o seguinte comando no Git Bash para aplicar as mudanças:)
java -version

Extensões do VSCode:
Java (Java Platform Extension for Visual Studio Code)
Java Extension Pack: Inclui todas as ferramentas necessárias para desenvolvimento em Java no VSCode.
Spring Boot Extension Pack: Para trabalhar com Spring Boot, uma das principais frameworks para desenvolvimento backend em Java.
GitLens: Para integração com GitHub e controle de versão

Configurar o VSCode para utilizar o Java
na tela do VSCode, digitar ctrl + , 
Na barra de pesquisa de configurações, digite java.configuration.runtimes.
Clique em Edit in settings.json para abrir o arquivo settings.json
No código do arquivo, acrescebnte o trecho abaixo de "java.configuration.runtimes": [
{
    "java.configuration.runtimes": [
        {
            "name": "JavaSE-17",
            "path": "C:/Users/marcus.silva/jdk-17/jdk-17.0.1+12",
            "default": true
        }
    ]
}
Salve o arquivo settings.json (Ctrl + S).
Feche e reabra o Visual Studio Code
java -version

Verificar se o VSCode usa o Java do sistema:
Abra o shell do sistema e digite para verificar:
echo $env:JAVA_HOME
echo $env:PATH
(se não retornar nada então instale conforme abaixo)
$env:JAVA_HOME = "C:\Users\marcus.silva\jdk-17\jdk-17.0.1+12"
$env:PATH += ";C:\Users\marcus.silva\jdk-17\jdk-17.0.1+12\bin"

#CRIAR UM PROJETO JAVA

No VSCode pressione Ctrl + Shift + P
digite na barra de procura "Java: Create Java Project"
***************************************************************************************
aparecerão as opções:
Maven: Maven é uma ferramenta de automação de compilação usada principalmente para projetos Java. Ele gerencia dependências, compilação, testes e distribuição de projetos Java. O Maven usa um arquivo de configuração chamado pom.xml (Project Object Model) para definir a estrutura do projeto, suas dependências e plugins.

Gradle: Assim como o Maven, o Gradle é outra ferramenta de automação de compilação. Ele oferece flexibilidade e desempenho superiores ao Maven. O Gradle também usa um arquivo de configuração, geralmente chamado build.gradle, para definir a estrutura do projeto e suas dependências.

Spring Boot: Spring Boot é um framework Java para criar aplicativos web rapidamente. Ele simplifica o desenvolvimento de aplicativos Spring, fornecendo configurações padrão e simplificando a configuração. Spring Boot é altamente modular e pode ser facilmente integrado com outras tecnologias Java.

Quarkus: Quarkus é um framework Java para construir aplicativos nativos da nuvem. Ele é otimizado para uso em contêineres e fornece inicialização rápida e baixo consumo de memória. Quarkus é particularmente adequado para microserviços e aplicativos em nuvem.

Micronaut: Micronaut é outro framework Java para construir aplicativos em nuvem. Ele é projetado para ser leve, rápido e altamente reflexivo. Micronaut oferece injeção de dependência eficiente, inicialização rápida e suporte a testes integrados.

GraalVM: GraalVM é uma máquina virtual universal que suporta várias linguagens de programação, incluindo Java. Ele oferece desempenho superior e menor consumo de recursos em comparação com outras JVMs tradicionais. GraalVM também pode ser usado para criar aplicativos nativos da nuvem.
